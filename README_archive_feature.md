# Функция архива растений

## Обзор
Добавлена функция архива для перемещения растений, которые больше не требуют активного отслеживания. Архивные растения скрываются из основного списка, но остаются доступными для просмотра и восстановления.

## Реализованные функции

### 1. Маршруты
- `/archive` - страница архива с отображением всех архивных растений
- `/move_to_archive/<int:plant_id>` - переместить растение в архив
- `/restore_from_archive/<int:plant_id>` - восстановить растение из архива

### 2. Шаблоны
- В `plant_detail.html` добавлена кнопка "Переместить в архив" в профиле растения
- В `plants.html` реализована фильтрация и отображение архивных растений
- В `base.html` добавлена навигация к архиву в главном меню

### 3. Модель данных
- В модели `Plant` добавлено поле `archived` (Boolean, по умолчанию False)
- В `models.py` поле уже реализовано в строке 78: `archived = db.Column(db.Boolean, default=False)`

### 4. SQL миграция
Файл `migrate_to_add_archive_final.sql` содержит скрипт для обновления существующей базы данных:
- Проверяет существование столбца `archived`
- Добавляет столбец при необходимости
- Создает индекс для улучшения производительности
- Устанавливает значение по умолчанию для существующих записей

## Использование

### Архивирование растения
1. Перейдите на страницу профиля растения
2. Нажмите кнопку "Переместить в архив"
3. Подтвердите действие в диалоговом окне

### Просмотр архива
1. Используйте ссылку "Архив" в главном меню
2. Или перейдите по адресу `/archive`

### Восстановление из архива
1. На странице архива найдите нужное растение
2. Нажмите кнопку "Восстановить"
3. Подтвердите действие в диалоговом окне

## Преимущества
- Уменьшает clutter в основном списке растений
- Сохраняет историю растений для будущих ссылок
- Позволяет легко восстановить растение при необходимости
- Поддерживает все данные растения, включая хронологию событий
